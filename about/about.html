Ryan Dahl on what he regrets about NodeJS

Ryan Dahl, the creator of Node.js, talks about the 10 things he regrets about Node.js. He also talks about the future of Node.js and how it will be used in the future. The talk was given at JSConf EU 2019 in Berlin, Germany. Things that Ryan regrets about Node.js:

1. Not sticking with Promises
2. Security issues
3. The Build System
4. package.json
5. node_modules
6. JavaScript Runtime
7. Bugs
8. Extensions
9. Autotools
10. Dynamic Languages
